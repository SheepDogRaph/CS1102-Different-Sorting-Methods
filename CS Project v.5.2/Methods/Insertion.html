Insertion Sort




Insertion sort is a direct grouping method that repeatedly puts the respective items(such as numbers ) of an unsorted list into the accurate order in a sorted portion of the list until it is correct in order. The number of the inputs are equal to the output.

The basic steps of the insertion sort algorithm are as follows:
Assume the first number is a, the second is b, the third is c, and the fourth is d…

Given that the insertion sort has two parts: sorted array(for the first number) and unsorted array(for the other number).

  1. The comparison between a and b in the list.
    2. If the b is less than a, then replace it with a.
  2. Comparing c,b, a, and exchanging the position between a,b, and c in the correct order(ascending or descending order based on the requirement) by comparing c to b,b to a.
  3. Repeat the steps by comparing a,b,c, and d, and exchange the order correctly based on ascending order or descending order based on the requirement.
  4. Repeat until all numbers are grouped.

Advantages:

  • Simplicity in a small amount of data: If the number of data is 100, insertion sort needs 0.003 which is the second least among these five sorting methods.
  • Easy to apply: Comparing the number with the previous order of the number.
  • Sort-stable: The number of the inputs are equal to the output.
  • Space-efficient: The space complexity of insertion sort is O(1).

Disadvantages:

  • Low efficiency in ordering larger sample size: If the number of the data is 10000, insertion sort needs to use almost 100 seconds to complete while quick sort might only need 0.4 seconds to complete.
  • The time usage is lower than quicksort: The time complexity of insertion sort is O(n) for the best cases while the quick sort is O(nlogn). The time complexity of insertion Sort is O(n²) for the worst and average cases, where ’n’ means the number of data in the list.

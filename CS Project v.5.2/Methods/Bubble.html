Bubble Sort




Bubble sort is an algorithm for sorting a list by iteratively comparing and swapping adjacent elements if they are in the incorrect order. This process continues until the entire list is arranged in the correct order.

The basic steps of the bubble sort algorithm are as follows:

  1. There is an initial unsorted array.
  2. Compare each pair of adjacent elements.
  3. Swap the elements if they are in the wrong order; otherwise, keep them the same.
  4. Move to the next pair of elements and repeat the steps above.
  5. End the sorting action when the corrected order list is obtained.

Advantages:

  • Efficient when Data is ‘Nicely’ Distributed
    The best-case time complexity of the algorithm is O(n), where no swapping is required. This situation will happen when the list is already in a sorted state or very close to being sorted.
  • Space Efficiency:
    Since bubble sort works in space, it does not need extra memory for variable storing and element swapping.

Disadvantages:

  • Dependent on Distribution:
    Selection sort is when there are many non-uniform elements. The worst-case and average-case time complexity of the algorithm is O(n2) when the list is in completely reverse order. In such cases, a significant number of swapping actions are required due to the large number of elements involved.
  • Slow performance:
    It takes a long time to finish sorting if there are many non-uniform elements. You can perceive the performance time from the animation

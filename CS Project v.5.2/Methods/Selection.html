Selection Sort




Selection sort is a sorting algorithm that is characterized by its iterative approach of repeatedly selecting the smallest (or largest) element from the unsorted portion of the list. Then, relocate it to its correct position by swapping it with the first element. The sorting action will keep going until the entire list is arranged in the correct order.

The basic steps of the selection sort algorithm are as follows:

  1. Start at the beginning of the initial unsorted array.
  2. Pick out the smallest element in the unsorted part.
  3. Swap it with the first element of the unsorted portion.
  4. Shift the boundary of the sorted portion one element to the right.
  5. Repeat the steps above and stop the sorting movement when the list is in the correct order.

Advantages:

  • Efficient when Data is ‘Nicely’ Distributed:
    Selection sort operates a better list that includes more uniform elements.
  • Space Efficiency:
    Since selection sort is working in space, it does not need extra memory for variable storing and element swapping.

Disadvantages:

  • Inefficiency
    The worst-case and average-case time complexity of the algorithm is O(n^2). Therefore, this sorting is not suitable when a large number of elements are involved.
  • Lack of Stability
    Since selection sort ​​swaps non-adjacent elements. when there are repeated elements in the input list, the relative order might be changed.
